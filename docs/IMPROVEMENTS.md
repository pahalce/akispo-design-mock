# AkiSpot デザインモックアップ改善提案

> **スコープ**: このドキュメントは、デザインモックアップとしての画面確認と改善に焦点を当てています。
> バックエンド連携、API実装、パフォーマンス最適化、セキュリティ対策などはReactでの実装フェーズで対応します。

---

## 📋 全体的な改善点

### 1. **デザインの一貫性**
- [ ] Material Symbolsのアイコンサイズが統一されていない（24px/48px混在）
- [ ] カードの角丸サイズの統一（一部2rem、一部1rem）
- [ ] シャドウの統一（shadow-sm/shadow-lg/shadow-xl）
- [ ] ボタンのpadding統一
- [ ] z-indexの管理ルール（モーダル=50、ナビ=20など）

### 2. **カラーパレット**
- [ ] グレーの色が複数ある（gray-400, gray-500, gray-600など） → デザインシステムとして定義
- [ ] ホバー時の色変化の統一（primary/90 or primary/80?）
- [ ] 非アクティブ状態の色を明確に定義
- [ ] disabled状態のスタイル定義

### 3. **タイポグラフィ**
- [ ] フォントサイズの階層を明確に（3xl, 2xl, xl, lg, base, sm, xs）
- [ ] line-heightの統一ルール
- [ ] font-weightの使い分けを明確に（400=通常、500=medium、700=bold）
- [ ] 英数字と日本語の混在時の見た目

### 4. **スペーシング**
- [ ] 余白の統一（gap, padding, marginの使い分けルール）
- [ ] セクション間のスペーシングルール
- [ ] カード内部のpadding統一

## 🔧 デザインシステムの整理

### コンポーネントの標準化
- [ ] **ボタン**: Primary / Secondary / Tertiary / Destructive
- [ ] **カード**: Default / Elevated / Outlined
- [ ] **入力フィールド**: Text / Select / Toggle
- [ ] **チップ/タグ**: Default / Selectable / Removable
- [ ] **ナビゲーション**: Bottom / Top
- [ ] **モーダル**: Full / Bottom Sheet / Alert

### アニメーション定義
- [ ] ページ遷移（fade / slide）
- [ ] モーダル表示（slide-up / fade-in）
- [ ] カード操作（swipe / flip）
- [ ] ボタン操作（scale / ripple）
- [ ] ローディング（spinner / skeleton）

### 状態の定義
- [ ] Default（通常）
- [ ] Hover（ホバー時）
- [ ] Active（アクティブ時）
- [ ] Pressed（押下時）
- [ ] Disabled（無効時）
- [ ] Loading（読み込み中）
- [ ] Error（エラー時）
